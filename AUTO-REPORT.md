# รายงานรายเดือนอัตโนมัติ (ส่งอีเมลทุกวันที่ 1)

ระบบจะส่งรายงาน **DOCX (พร้อมกราฟ)** ทางอีเมลอัตโนมัติทุกวันที่ 1 ของเดือน
โดยใช้ข้อมูล **ตั้งแต่วันที่ 1 ถึงสิ้นเดือนของเดือนก่อนหน้า**

## หลักการทำงาน
- ใช้ **GitHub Actions** (cron) เปิด Chromium จริงแบบ headless → เข้าเว็บแอป → ล็อกอินบัญชีบอท → เรียกฟังก์ชัน `sendAutoMonthly()` ของเว็บ
- จึงได้รายงานเหมือนกดส่งเอง 100% (มีกราฟ + ตารางครบ) ส่งผ่าน Brevo (ตัวเดิม)
- ไฟล์ที่เกี่ยวข้อง: `.github/workflows/monthly-report.yml`, `.github/scripts/send-monthly-report.mjs`

## ตั้งค่าครั้งเดียว

### 1) สร้างบัญชี "บอท" สำหรับรันรายงาน
ล็อกอิน admin → **จัดการผู้ใช้ระบบ** → เพิ่มผู้ใช้ เช่น
- Email: `report-bot@somapait.com` (หรืออีเมลใดก็ได้ที่คุณคุม)
- รหัสผ่าน: ตั้งให้ปลอดภัย
- สิทธิ์: **senior** (พอสำหรับอ่านข้อมูล+ส่งรายงาน)

> บัญชีนี้ต้องผ่านการอนุมัติ (มี role) จึงล็อกอินได้

### 2) ใส่ค่า Secrets ที่ GitHub repo
ไปที่ repo `Nattha-OSO/OSO-Evaluation-by-IMM` → **Settings → Secrets and variables → Actions → New repository secret** เพิ่ม 3 ตัว:

| ชื่อ Secret | ค่า |
|---|---|
| `REPORT_BOT_EMAIL` | อีเมลบัญชีบอท (ข้อ 1) |
| `REPORT_BOT_PASSWORD` | รหัสผ่านบัญชีบอท |
| `REPORT_RECIPIENTS` | อีเมลผู้รับรายงาน (หลายคนคั่นด้วย , ) |

> (ไม่บังคับ) ตั้ง **Variable** ชื่อ `APP_URL` หากเปลี่ยน URL เว็บ — ค่าเริ่มต้นคือ GitHub Pages ปัจจุบัน

### 3) เปิดใช้งาน + ทดสอบ
- ตรวจว่า **Brevo** ถูกตั้งค่าแล้ว (เหมือนการส่งรายงานปกติ — `BREVO_API_KEY`, `SENDER_EMAIL` ใน Supabase)
- ทดสอบทันที: repo → แท็บ **Actions → Monthly Report (Auto Email) → Run workflow**
- ดู log ว่าขึ้น `✅ ส่งรายงาน ... ไปยัง: ...`

## กำหนดเวลา
- ค่าเริ่มต้น: ทุกวันที่ 1 เวลา **08:00 (ไทย)** = `cron: '0 1 1 * *'` (UTC)
- แก้เวลาได้ในไฟล์ `.github/workflows/monthly-report.yml`
- หมายเหตุ: GitHub อาจรัน cron ช้ากว่ากำหนดเล็กน้อย (ปกติของ GitHub Actions)

## หมายเหตุ
- การส่งถูกบันทึกใน Audit Log (ในชื่อบัญชีบอท)
- ถ้าเดือนก่อนหน้าไม่มีข้อมูล ระบบยังส่งรายงาน (แสดง "ไม่มีข้อมูล" ในตาราง)
- ต้องการความถี่อื่น (รายสัปดาห์/รายไตรมาส) แก้ cron + ช่วงวันที่ในฟังก์ชัน `sendAutoMonthly` ได้
